1. Скачайте архив (там будет несколько файлов: 3 php-файла для парсинга курса обмена с бирж (BTC-E, Bitstamp, Huobi), js скрипт, файл СSS, и эта инструкция).

2. Выберете источник обмена (т.е. - биржу) и загрузите соответствующий файл парсера котировок в корневую папку Вашего сайта. Для биржи BTC-E.com это будет vp-btce.php, для Bitstamp.com - vp-bitstamp.php, и vp-huobi.php для Huobi.com.

3. Настройте скрипт замены, файл scrypt.js - пропишите исходную валюту вашего сайта (т.е. ту, из которой надо сделать пересчёт в bitcoin, строка 19), биржу (строка 20), и интервал обновления (строка 22) в милисекундах.
   - из доллара ФРС bitcoin можно пересчитать по курсу BTC-E или Bitstamp.
   - euro считается исходя из внутреннего курса доллар ФРС/euro на бирже Bitstamp.
   - юань пересчитывается по кусу биржи Huobi.

4. В коде страниц, на которых нужно делать пересчёт, поместите содержимое этого файла (scrypt.js) cразу после первого тега <body>.

5. В заголовках страниц (между тегами <head> и </head>), на которых нужно делать пересчёт, поместите следующую строку:
<script type="text/javascript" src="http://code.jquery.com/jquery-latest.min.js"></script>

6. В файл CSS (который описывает дизайн требуемых Вам страниц) добавьте содержимое файла style.css, настроив внешний вид "кнопок" переключения валют (изначально их id -"start_convert" и "restart_convert") в соотвествии с дизайном Вашего сайта. 

7. Вы интегрировали скипт в страницы для работы.
Теперь, для того что бы компьютеру, а точнее - браузеру пользователя было ясно, где должна произойти замена значений, все пересчитываемые, исходные цены в коде страниц надо поместить (если они до этого не были туда помещёны) в тег <span> с классом "price", как в примере: 

было (в коде): Цена: 1000 руб. 
стало (в коде): Цена: <span class="price">1000 руб.</span>

8. !!! Чтобы пользователи увидели, что цена указается именно в bitcoin, а не непонятно в чём, поместить в тэг <span> надо и число, и обозначение валюты, что бы оно тоже было заменено!

9. Если же у Вас много мест, где надо заменить значение, и для указания цены уже используется другой класс, а не класс "price", можно прописать нужный класс в самом скрипте; строка 23 исходного файла scrypt.js.

10. Что бы учесть дополнительные затраты, неизбежные при конвертации (вводе и выводе) валют, добавьте соответствующий объёму дополнительных затрат коэффициент в 12 строке файла scrypt.js
(10% - это 1.1, 20% - это 1.2 , и т.д.):
было : 	new_val = (old_val / kotirovka).toFixed(4);
стало : new_val = (old_val / kotirovka*1.1).toFixed(4);

ВАЖНО: Скрипт несёт информационный характер! Проверяйте обменный курс при окончательном совершении сделок!!! 
В источниках данных для парсинга, которые использует скрипт, возможны изменения, которые могут сделать работу скрипта некорректной!
Так же, курс может значитльно измениться с момента обновления.

Владелец сайта, со своей стороны, обещает по возможности наиболее оперативное изменеие скрипта в вышеуказанном случае.
Для оповещения об этом, заполните контактную форму при загрузке.
(Вы можете не заполнять её, но тогда мы не сможем сообщить Вам об изменениях в скрипте!)
